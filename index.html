目的とゴール

目的とゴール

本資料は、AI音声対話モジュールの「試作第1号（MVP）」に向けた要件・構成・進行の共通理解を整えるためのものです。

ゴール

  • BLE接続＋音声入出力の安定動作
  • サイズ・重量・稼働時間の成立条件を確認
  • 量産前提の設計方針（拡張ポイント）を明確化

成果物（MVP）

  • 試作機（1台）＋回路図／BOMラフ
  • BLE GATT設計（テキスト送受信用）
  • 簡易動作ツール（疎通検証用）
2 / 12
主要要件（公募要約）

主要要件（公募要約）

  • BLE通信（スマホとペアリング／テキスト送受信）
  • MEMSマイク入力
  • 小型スピーカー出力（0.5–1Wクラス）
  • 基板サイズ：50×40mm以内／厚さ20mm以下
  • 連続稼働：3–6h 目安
  • 簡易3Dプリント筐体（脱着構造）

※ AI処理はアプリ側で担当（モジュール側は非対応）

3 / 12
追加要件（先方仕様）

追加要件（先方仕様）

  • サイズ：70×40×15mm ／ 重量：≤60g
  • 動作時間：連続7時間以上
  • LED表示（電源／通信）
  • GPS位置情報
  • タッチセンサー
  • OTAアップデート、起動時オフラインチェック
  • 音声→テキスト→AI→音声合成（TTS）
  • 量産計画：初回500台 → 年間3,000台

MVP構成（ハード）

MVP構成（ハード）

主要部品

  • MCU：ESP32-S3（BLE＋低消費電力）
  • 音声：I2S MEMSマイク
  • アンプ：1W級クラスD＋小型SP
  • 電源：LiPo 1200–2000mAh＋充電IC（USB-C）

筐体

  • 3Dプリント（スナップ式 or ビス固定）
  • 放熱と音量のバランス設計
  • LED/GPS/タッチ用の実装余白を確保

MVP構成（ファーム/IF）

MVP構成（ファーム / インタフェース）

  • BLE GATT：テキスト送受信用サービス／再接続戦略
  • 省電力：スリープ制御／広告間隔最適化
  • テスト用ツール：疎通確認（Web/スクリプト）
  • 将来拡張：LED/GPS/タッチ/OTAのフック点を用意

アーキテクチャ（概念図）

アーキテクチャ（概念図）

スマホアプリ

  • BLEペアリング
  • テキスト送受信
  • 音声→テキスト（Whisper等）
  • AI応答→TTS

モジュール（MVP）

  • ESP32-S3
  • I2S MEMS Mic / 1W Amp
  • LiPo / USB-C 充電
  • 拡張：LED / GPS / タッチ / OTA

※ AI対話処理はアプリ側。MVPは堅牢な入出力と通信の土台を提供。

技術的チャレンジと手当

主要リスク

  • 7h稼働とサイズ/重量のトレードオフ
  • 1W駆動時の放熱・音量確保

  • 1200/1500/2000mAhの容量選択肢
  • 高効率アンプ／通気設計／消費電流監視
  • 広告間隔/再接続の最適化とリカバリ
